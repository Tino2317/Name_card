After obtaining my CCNA certification, I decided to combine and utilize the knowledge I gained from the certification together with the knowledge I obtained from the python course I studied on Udemy.

I then created labs on Cisco Packet Tracer and GNS3 to gain as much hands on practical experience that's close to the real world working environment.

I integrated GNS3 with a virtual machine called VMware Workstation Pro. The virtual machine is essential because it allows you to work on GNS3 projects independent of your computers operating system, therefore increasing performance in the labs.

EIGRP routing. Default route and passive-interface.

Slides:
1. Topology diagram. I preconfigured all router interfaces and PC's with IP addresses. Each router interface is in a different subnet.

2. Router R1 shows its interfaces. I then configured an EIGRP routing protocol on all routers with an autonomous system number of 100. The network advertised is 10.0.0.0 with a classful default widcard mask of 0.255.255.255.

3. I configured a default static route on router R4 which goes to the internet and redistributed it into the EIGRP routing protocol. I also made interface f1/1 on R4 a passive interface so that internal information is not shared to the internet.

4. The routing table of router R4

5. The routing table of router R1 and the learnt default static route "D*EX".

6. PC1 on subnet 10.0.1.0 pings PC3 on subnet 10.1.2.0 and the ping is successful.

Single Area OSPF. Reference bandwidth, OSPF cost, passive interface and default route.

Slides:
1. Topology diagram. I preconfigured all router interfaces and PC's with IP addresses. Each router interface is in a different subnet.

2. I configured loopback interfaces and an OSPF routing protocol on all routers. The "show ip ospf neighbor" command on R1 shows that adjacencies formed between R1 and its adjacent routers.

3. The routing table on router R3 shows all the learnt routes through OSPF.

4. I changed the reference bandwidth on all routers to 100000, so that when faster interfaces like 100Gbps are implemented, they will have a cost of 1 and have first preference.

5. For load balancing, I changed the cost for traffic moving from R1 to the 10.1.2.0 network on R4 to be equal through R2 and R5. Previously traffic from R1 only passed through R2.

6. On R4, I configured interface f1/1 to be a passive interface and advertise network 203.0.113.0 through OSPF. I also configured a default static route which I redistributed into OSPF using the " default-information originate" command.

7. Routing table of R4, showing the default static route.

8. Routing table for R2 showing the learnt default static route "O*E2".

9. I pinged PC2 on the 10.0.2.0 network from PC3 on the 10.1.2.0 network. The ping was successful.

Multi-Area and Multi-Access OSPF networks. Summarisation, ABR's, DR and BDR.

Slides:
1. Routers are placed in different OSPF areas. R4 and R3 are in area 0 and R1 is in area 1. R2 and R5 are Area Border Routers (ABR's).

2. OSPF is enabled on all routers and adjacencies are formed.

3. Routing table on R1 shows the inter area routes, "O IA".

4. Route summarisation is performed on the two ABR's, R2 and R5.

5. R1 routing table now only shows 1 summarised route to other networks.

6. OSPF Multi-access network Topology.

7. All routers are configured for OSPF routing.

8. The "show ip ospf neighbor" command on R6 shows that R9 is the Designated Router(DR) and R8 is the Backup Designated Router(BDR).

9. I made R6 the DR by giving it a higher OSPF priority number than the other routers.

10. The "show ip ospf int f0/0" on R6 shows that R6 is the DR now and R8 remains the BDR.

HSRP, STP, NAT, Access-lists. Pre-emption and Translations.

Slides:
1. Topology consists of service provider routers(SP1 and SP2), edge routers(R1 and R2), collapsed core and distribution switches(CD1 and CD2), access layer switches(Acc3 and Acc4), and two PC's.

2. Configured R1 and R2 with the virtual IP address. R2 became the active router and R1 is the standby router.

3. I then made R1 the active router by giving it a higher HSRP priority number than R2. The pre-emption command allows R1 to stay active when its functioning.

4. I made CD1 the root bridge so it can align with R1, the HSRP active gateway.

5. I disabled the Spanning Tree Protocol(STP) on all switchports where a single end host is connected.

6. I pinged the virtual IP address from PC1, It was successful.

7. Network Address Translation(NAT) topology.

8. I configured R1 with Dynamic NAT overload.

9. PC2 pinged an IP address on the internet, It was successful.

10. The debugging command allows us to see the translations on R1 from PC2.

Etherchannel and Port-security. LACP and PAgp.

Slides:
1. Topology diagram consists of access layer switches(Acc3 and Acc4), collapsed distribution and core layer switches(CD1 and CD2) and PC's.

2. I configured links between Acc3, CD1, and CD2 with the Link Aggregation Control Protocol(LACP) etherchannel.

3. The "show etherchannel summary" command shows the port-channels created which provide fault tolerance, high overall bandwidth, redundancy and load balancing.

4. The links between Acc4, CD1 and CD2, I configured the Port Aggregation Protocol(PAgP) etherchannel.

5. Acc4 shows the port-channels created by PAgP which have the same benefits as LACP.

6. Simple network topology of a switch and two PC's

7. I disabled all unused ports on SW1. This prevents unauthorised hosts from plugging into them to gain access to the network.

8. I pinged PC2 from PC1 to generate a mac-address table on SW1.

9. I then configured the two functioning ports with port-security.

10. The "show port-security command" verifies the port-security configurations on both ports.
